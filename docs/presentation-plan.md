# 発表資料 構成計画

## 基本情報

- **発表時間**: 10〜15分（12分発表 + 3分質疑を想定）
- **スライド枚数**: 15〜18枚
- **方向性**: AI活用プロセスを軸に、審査員が高得点をつけざるを得ない構成
- **形式**: スライド + ライブデモ or デモ動画

## 審査の評価軸（しおりP7より）

1. **サービスとしての価値**: 役に立つ / 面白い / 使いたいと思えるか
2. **完成度**: 動作するか、UI/UXのこだわり
3. **技術的な挑戦**: 新しい・難しい技術に挑戦しているか
4. **生成AIを使いこなせているか**

## 発表資料の必須項目（しおりP6より）

- なぜこのプロダクトを作ったか（きっかけ・背景）
- 生成AIをどのように活用したか
- 実際の画面・動作の様子（デモまたは動画）
- こだわった点・工夫した点
- 今後の展望・改善点
- 開発時どこで苦労したか

## 差別化ポイント

### スライド6「AI活用マップ」が最大の武器
他チームは「ChatGPTでコード書きました」程度になりがち。我々は「企画（Claude）→ 設計（Claude Code Max）→ 実装（Claude Code Max + MCP）→ アセット（Midjourney/Suno/Udio）→ テスト（CI/CD自動）→ タスク管理（Notion）」と開発ライフサイクル全体でAIを組み込んでいることを1枚の図で見せる。

### スライド9「Unity MCP連携」
AIがコードを生成するだけでなく、Unity Editorを直接操作する「AIエージェント開発」は先端的。M3の技術役員（山崎様・河合様）にも刺さる。

### デモで「完成度」を証明
動くものを見せることが最強の説得力。END1（最短）とバトル→分岐のデモで「ちゃんと動く」ことを見せる。

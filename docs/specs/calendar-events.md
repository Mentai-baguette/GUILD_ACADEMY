# カレンダー・イベント管理 仕様書

## 1. ゲーム全体の時間構造

### 1-1. 期間概要

2年制の学園（ただしゲーム内は実質2年分の4章構成）。

| 期間 | 内容 | 週数 |
|------|------|------|
| 1章 | 1年目 前期 | 12週 |
| 夏休み① | 1年目 夏期休暇 | 2週 |
| 2章 | 1年目 後期 | 12週 |
| 冬休み | 冬期休暇 | 3週 |
| 3章 | 2年目 前期 | 12週 |
| 夏休み② | 2年目 夏期休暇 | 2週 |
| 4章 | 2年目 後期 | 12週 |
| **合計** | | **57週** |

> NG+ではカレンダーは完全リセットされる。

### 1-2. 1週間の構造

| 曜日 | 区分 |
|------|------|
| 月〜金 | 平日（授業あり） |
| 土曜 | 終日自由行動 |
| 日曜 | 特殊イベント枠 |

## 2. 1日の時間帯構造

### 2-1. 平日

```
[朝] → [授業(1〜4コマ)] → [5コマ目(任意)] → [放課後] → [夜]
```

| 時間帯 | 概要 | 自由行動 |
|--------|------|---------|
| 朝 | 起床〜授業開始。強制イベント/朝会話あり | 不可（イベント消化のみ） |
| 授業 | 1〜4コマ（必須）+ 5コマ目（任意） | 不可（サボりは可） |
| 放課後 | 授業終了後〜夕方。メインの自由行動時間 | 可（5コマ目ありなら不可） |
| 夜 | 夕方〜就寝。寮での活動 | 可（制限あり：ダンジョンは+1層のみ） |

### 2-2. 土曜日

```
[朝] → [終日自由行動] → [夜]
```

| 時間帯 | 概要 | 自由行動 |
|--------|------|---------|
| 朝 | 起床。強制イベントがあれば消化 | 限定的 |
| 日中 | 終日自由行動。ダンジョン・SL・探索等 | 可 |
| 夜 | 寮活動。会話・イベント | 可 |

### 2-3. 日曜日

```
[朝] → [特殊イベント枠 or 自由行動] → [夜]
```

- 特殊イベント（遠征・大会・祭り等）が配置される優先曜日
- 特殊イベントがない週は自由行動と同じ

### 2-4. 長期休暇

```
[朝] → [全パート自由行動] → [夜]
```

- 授業なし。全時間帯が自由行動
- 強制ストーリーイベントが数日間配置される
- 全員帯同可能（授業による制約なし）

## 3. イベント種類と優先度

### 3-1. イベント種類

| 種類 | 優先度 | 説明 | スキップ |
|------|--------|------|---------|
| 強制イベント | 最高(100) | メインストーリーの分岐点、加入イベント等 | 不可 |
| メインストーリー | 高(80) | 章の進行に必須だが日程に幅がある | 不可 |
| ソウルリンク(SL) | 中(60) | キャラとの絆イベント。条件を満たすと発生 | 可（見逃すとその章で消失） |
| 教師イベント | 中(50) | 教師NPCとの信頼度イベント | 可（次章に持ち越し不可） |
| ランダムイベント | 低(30) | 日常系・小ネタ・サブクエスト | 可（再発生あり） |
| 自由行動 | 最低(10) | プレイヤーが自由に行動を選択 | — |

### 3-2. 優先度ルール

- **強制イベントは割り込み**: 他のイベントが予定されていても、強制イベントが優先
- 同一優先度の場合はイベントID順（先に登録されたものが優先）
- SLイベントと教師イベントが同日に発生可能な場合、プレイヤーが選択

## 4. 時間帯別イベント配置ルール

| 時間帯 | 配置されるイベント種類 |
|--------|---------------------|
| 朝 | 強制イベント, 授業関連イベント, 朝の寮会話 |
| 授業中 | 授業イベント（教師信頼度関連）, サボりイベント |
| 放課後 | キャラ加入イベント, SLイベント, ダンジョン探索, 自由行動 |
| 夜 | 侵蝕浄化(ユナ), 寮会話イベント, シリアス系イベント, 夜間ダンジョン(+1層) |
| 土日日中 | 遠征, 闘技場大会, 学園祭, 大規模イベント |

## 5. 章別イベントスケジュール概要

### 5-1. 1章（1年目前期・12週）

| 週 | 主要イベント |
|----|------------|
| 1週 | 入学式(強制), レイ・ユナ・ミオ操作開始, 学園探索チュートリアル |
| 2週 | カイト加入イベント(放課後), 最初のダンジョン「翠風の森」解放 |
| 3週 | リン加入イベント(放課後), 戦闘チュートリアル完了 |
| 4週 | ジン加入イベント(放課後), SLシステム解放 |
| 5週 | シオン登場イベント(朝), 先輩としての初接触 |
| 6週 | ヴェイン加入イベント(放課後), 教団の噂イベント |
| 7週 | メル加入イベント(放課後), 風鳴りの塔解放 |
| 8週 | レンジ加入イベント(放課後) |
| 9週 | セツナ加入イベント(放課後), 氷結洞窟解放 |
| 10週 | カルロスの特別授業(強制), 情報フラグ関連ヒント |
| 11週 | 1章ボスダンジョン解放 |
| 12週 | 章末テスト(強制), 1章ボス撃破期限 |

### 5-2. 夏休み①（2週）

- 自由行動中心。ダンジョン周回チャンス
- レンジの実家訪問イベント（任意）
- 強制: 夏の特訓イベント（1日のみ）

### 5-3. 2章（1年目後期・12週）

| 週 | 主要イベント |
|----|------------|
| 1週 | 後期開始(強制), 新ダンジョン解放, 履修再登録 |
| 3週 | シオンとの共同任務(強制), シオンSL進行 |
| 5週 | 闘技場イベント(日曜), 大会参加 |
| 7週 | カルロスの不審な行動イベント(夜) |
| 9週 | 教団エクリプスの襲撃(強制), カルロスの正体発覚 |
| 10週 | シオンの決意イベント(強制) |
| 11週 | **シオン離脱**(強制)。装備全返却 |
| 12週 | 章末テスト(強制), 2章ボス撃破期限 |

### 5-4. 冬休み（3週）

| 週 | 内容 |
|----|------|
| 1週 | 帰省イベント。ユナ/ミオの実家に遊びに行ける（選択） |
| 2週 | 自由行動 + 強制ストーリー（教団の動き） |
| 3週 | 年明けイベント(強制) + 自由行動 |

**冬休み特殊ルール**:
- シオンの部屋に入れない（離脱中のため）
- ユナの実家: ユナSL+信頼度イベント
- ミオの実家: ミオSL+幼少期回想イベント
- 寮に残る場合: 他メンバーとの交流イベント

## 6. イベントデータ構造

### 6-1. ScriptableObject 形式

```csharp
[CreateAssetMenu(fileName = "NewEvent", menuName = "GA/Event/EventData")]
public class EventData : ScriptableObject
{
    [Header("基本情報")]
    public string eventId;            // 一意のID (例: "ch1_w2_kaito_join")
    public string eventName;          // 表示名
    [TextArea] public string description;

    [Header("配置")]
    public int chapter;               // 1〜4
    public int week;                  // 1〜12 (休暇中は特殊値)
    public DayOfWeek dayOfWeek;       // 曜日指定 (Any = 任意)
    public TimeSlot timeSlot;         // Morning / Class / AfterSchool / Night / AllDay
    public VacationType vacation;     // None / Summer1 / Winter / Summer2

    [Header("優先度・種類")]
    public EventType eventType;       // Forced / Main / SoulLink / Teacher / Random / Free
    public int priority;              // 10〜100

    [Header("発生条件")]
    public EventCondition[] conditions; // フラグ・信頼度・章等の条件配列

    [Header("結果")]
    public EventResult[] results;     // フラグ変更・信頼度変動・アイテム獲得等

    [Header("演出")]
    public string sceneId;            // 会話シーンID（ダイアログスクリプト参照）
    public bool isSkippableOnNG;      // NG+で既読スキップ可能か
}
```

### 6-2. Enum 定義

```csharp
public enum TimeSlot
{
    Morning,      // 朝
    Class,        // 授業中
    AfterSchool,  // 放課後
    Night,        // 夜
    AllDay,       // 終日（土日・休暇用）
    Any           // 時間帯不問
}

public enum EventType
{
    Forced,       // 強制（最高優先度）
    Main,         // メインストーリー
    SoulLink,     // ソウルリンク
    Teacher,      // 教師信頼度
    Random,       // ランダム日常
    Free          // 自由行動
}

public enum VacationType
{
    None,         // 通常学期
    Summer1,      // 夏休み①
    Winter,       // 冬休み
    Summer2       // 夏休み②
}
```

### 6-3. 条件データ

```csharp
[System.Serializable]
public class EventCondition
{
    public ConditionType type;    // Flag / Trust / Chapter / SoulLink / Item / Day
    public string targetId;       // フラグ名 / キャラ名 etc.
    public CompareOp op;          // Equal / GreaterThan / LessThan
    public int value;
}
```

## 7. イベントスケジューラの処理フロー

```
毎日の開始時:
  1. 現在の章・週・曜日・時間帯を取得
  2. 全イベントの発生条件をチェック
  3. 条件を満たすイベントを優先度順にソート
  4. 強制イベントがあれば割り込み実行
  5. 残りのイベントをプレイヤーに提示（選択肢として）
  6. プレイヤーが行動を選択 or 自由行動
  7. イベント結果を適用（フラグ・信頼度・アイテム等）
  8. 次の時間帯へ遷移
```

---

## 8. 実装上の注意

- イベントの発生可否は毎時間帯の開始時に再評価する（途中で条件が変わる場合あり）
- 強制イベントが複数重なった場合はpriority値で解決し、低い方を翌日に繰り越す
- SLイベントは「その章内で見逃すと消失」するため、発生可能日のカウントダウンUIを表示する
- NG+では `isSkippableOnNG` が `true` のイベントに「スキップ」ボタンを表示

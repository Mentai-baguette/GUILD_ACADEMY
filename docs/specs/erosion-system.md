# 侵蝕（Erosion）システム仕様書

## 概要

主人公レイ専用のリスク＆リワードシステム。闇の力を使うほど侵蝕値が上昇し、高い攻撃力を得る代わりに暴走のリスクが増大する。侵蝕値はエンディング分岐にも影響する。

---

## 1. 侵蝕段階

### 通常時（エルト撃破前）

| 段階 | 名称 | 侵蝕値範囲 | ATK倍率 | 暴走リスク |
|------|------|-----------|---------|-----------|
| 1 | Normal | 0% ~ 24% | x1.0 | なし |
| 2 | Unstable | 25% ~ 49% | x1.3 | なし |
| 3 | Dangerous | 50% ~ 74% | x1.6 | なし |
| 4 | Critical | 75% ~ 100% | x2.0 | あり |

### エルト撃破後（許容値上昇）

| 段階 | 名称 | 侵蝕値範囲 | ATK倍率 | 暴走リスク |
|------|------|-----------|---------|-----------|
| 1 | Normal | 0% ~ 34% | x1.0 | なし |
| 2 | Unstable | 35% ~ 59% | x1.3 | なし |
| 3 | Dangerous | 60% ~ 84% | x1.6 | なし |
| 4 | Critical | 85% ~ 100% | x2.0 | あり |

> エルト（封印の間15層最深部のボス）を撃破すると、レイの封印が一部安定化し、各段階の閾値が緩和される。

---

## 2. ATK倍率の適用

- 侵蝕ATK倍率はレイの全ての物理攻撃・闇魔法攻撃に適用
- 通常攻撃にも適用（闇属性が乗るため）
- 光属性・回復スキルには適用されない
- 計算式への適用位置:

```
最終ダメージ = (ATK x 侵蝕ATK倍率 x 2 - DEF) x スキル倍率 x 属性相性 x クリティカル x ブレイク x 乱数
```

---

## 3. 侵蝕値の上昇

### 闇スキル使用時

各闇スキルに「侵蝕上昇値」が個別設定される。

| スキルカテゴリ | 侵蝕上昇値 | 備考 |
|-------------|-----------|------|
| 通常攻撃（闇付与時） | +1% | 覚醒の道スキルで闇付与中 |
| 闇撃スキル Tier1-2 | +2% | 基本闇物理 |
| 闇撃スキル Tier3-4 | +3% | 中級闇物理 |
| 闇撃スキル Tier5 | +5% | 上級闇物理 |
| 闇魔スキル Tier1-2 | +2% | 基本闇魔法 |
| 闇魔スキル Tier3-4 | +4% | 中級闇魔法 |
| 闇魔スキル Tier5 | +6% | 上級闇魔法 |
| 覚醒の道スキル | +3% ~ +8% | 各スキルで異なる |
| 魔王の道（隠し路線）スキル | +5% ~ +15% | 最強だが侵蝕大 |
| デュアルアーツ（闇含む） | +10% | レイ+ヴェイン「深淵共鳴」等 |

### デュアルアーツの侵蝕

- レイが参加するデュアルアーツで闇属性が含まれる場合、+10%
- レイ不参加のデュアルアーツでは侵蝕上昇なし

### 被ダメージ時

- レイが闇属性攻撃を受けた場合: +2%
- その他の被ダメージ: 侵蝕上昇なし

## 4. 暴走（Berserk）

### 発動条件

1. 侵蝕段階が**Critical**であること
2. レイが**闇スキルを使用**すること
3. 暴走確率判定に成功すること

### 暴走確率

```
暴走確率 = (侵蝕値 - Critical閾値) x 2%
```

- 通常時: 侵蝕75%で0%、100%で50%
- エルト撃破後: 侵蝕85%で0%、100%で30%

### 暴走中の動作

| 項目 | 詳細 |
|------|------|
| 操作 | 完全に操作不能 |
| 行動 | 毎ターン自動で闇属性全体攻撃（味方含む） |
| 攻撃対象 | ランダム（敵味方問わず） |
| ダメージ | ATK x 侵蝕ATK倍率(2.0) x 1.5（暴走補正） |
| 持続 | 3ターン |
| 終了後 | 侵蝕値 -10%、HP50%、MP0 |

### 暴走の防止

- ユナのバトル中浄化スキルで侵蝕段階を下げておく
- Critical段階で闇スキルを使わない
- メルの歌による侵蝕軽減効果（侵蝕上昇値を50%カット、パッシブ/スキル）

---

## 5. 浄化（Purification）

### バトル中浄化（ユナ専用）

| 項目 | 詳細 |
|------|------|
| 使用者 | ユナのみ（浄化の道スキル） |
| 使用回数 | 1バトルにつき1回 |
| 効果 | 侵蝕段階を1段階下げる |
| 侵蝕値変動 | 現在の段階の下限値まで低下 |
| ターン消費 | あり（ATBゲージ全消費） |
| MP消費 | 30MP |

- 例: 侵蝕52%（Dangerous）→ 浄化 → 25%（Unstableの下限）
- Normal段階（0~24%）で使用した場合は0%にリセット
- エルト撃破後は閾値が変わるため、浄化後の値も変わる

### フィールド浄化（寮夜パート）

| 項目 | 詳細 |
|------|------|
| 発生 | 寮夜パートでユナイベントを選択 |
| 効果 | 侵蝕値 -15% |
| 消費 | 寮夜パートの行動1回分 |
| 回数制限 | 1夜につき1回 |
| 発生条件 | ユナが寮にいること |

### メルの侵蝕軽減

- メルの歌スキル（魂歌の道/癒歌の道の一部）がバトル中に発動中:
  - レイの闇スキル使用時の侵蝕上昇値を50%カット
  - 例: +6%のスキル → +3%に軽減

---

## 6. 自然回復

**侵蝕は自然回復しない。** 浄化手段のみで低下する。

- バトル終了後も侵蝕値は維持
- フィールド移動中も侵蝕値は維持
- セーブ/ロードで侵蝕値は保存される
- NG+開始時に侵蝕値は0%にリセット

---

## 7. 侵蝕とエンディングの関係

### END6条件

- 侵蝕値90%以上の状態でシオン第2形態戦に突入 → END6「全滅エンド」確定

### EndingResolverとの連携

既存コード（`EndingResolver.cs`）にて:
```csharp
if (context.ErosionPercent >= 90)
    return EndingType.TotalDefeat;
```

### 侵蝕とストーリー分岐

| 侵蝕段階 | ストーリーへの影響 |
|----------|------------------|
| Normal | 特になし |
| Unstable | 一部NPC台詞変化。レイの目の描写変化 |
| Dangerous | 仲間の心配イベント発生。一部NPCが恐怖を示す |
| Critical | 暴走リスク。グレイヴの警告イベント。シオン戦前にカットシーン変化 |

### END5への影響

- END5ルートでは侵蝕値が低いほどカルロス戦が有利
- 侵蝕0%でカルロス戦突入: レイの全ステ+10%ボーナス
- 侵蝕50%以上でカルロス戦: ペナルティなしだがボーナスもなし

## 8. 侵蝕の視覚表現

### UIゲージ

- バトルUI上部にレイの侵蝕ゲージを常時表示
- 色変化:
  - Normal: 緑
  - Unstable: 黄
  - Dangerous: 橙
  - Critical: 赤（点滅）

### キャラ描写

| 段階 | レイの外見変化 |
|------|-------------|
| Normal | 通常の外見 |
| Unstable | 左手の封印刻印が微かに発光 |
| Dangerous | 髪の毛先に紫がかった光。目の色が時折赤紫に |
| Critical | 体から黒い霧。両目が赤紫。封印刻印が脈動 |
| 暴走中 | 両目が漆黒。体全体に黒い亀裂。声が変質 |

## 10. 実装メモ

### データ構造

```
ErosionSystem
├── CurrentValue (float: 0.0 ~ 100.0)
├── IsEltDefeated (bool) — 閾値変更フラグ
├── CurrentStage → Normal/Unstable/Dangerous/Critical
├── AtkMultiplier → 1.0/1.3/1.6/2.0
├── PurifyCountThisBattle (int) — バトル中浄化使用回数
│
├── AddErosion(float amount) — 闇スキル使用時に呼び出し
├── Purify(int stages) — 浄化スキル使用時
├── PurifyField() — フィールド浄化(-15%)
├── CheckBerserk() → bool — 暴走判定
├── GetStageThresholds() → int[] — 現在の閾値配列
└── Reset() — NG+用リセット
```

### 閾値定数

```csharp
// エルト撃破前
static readonly int[] NormalThresholds = { 0, 25, 50, 75 };
// エルト撃破後
static readonly int[] PostEltThresholds = { 0, 35, 60, 85 };
```

### セーブデータ

- `ErosionValue` (float): 現在の侵蝕値
- `IsEltDefeated` (bool): エルト撃破フラグ
- `BerserkCount` (int): 暴走回数（実績用）

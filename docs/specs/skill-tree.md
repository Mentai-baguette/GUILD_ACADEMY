# スキルツリーシステム仕様書

## 概要

各キャラクターが固有のスキルツリーを持ち、SPを消費してスキルを習得する。路線（ブランチ）ごとにスキルが5段階あり、ビルドの多様性を確保する。

---

## 1. SP（スキルポイント）入手

### 入手源

| 入手方法 | 数量 | 備考 |
|----------|------|------|
| レベルアップ | 2Lvごとに1SP | 最大Lv100 → 最大50SP |
| テスト好成績 | 各章末テストで最大2SP | 全4章 x 2SP = 最大8SP |
| イベント | 約42SP | サブイベント・メインストーリー報酬 |
| **合計** | **約100SP** | |

### NG+累積ボーナス

- 毎周 +10SP（累積）
- 2周目: +10SP、3周目: +20SP、...
- 上限なし

---

## 2. スキルツリー構造

### 路線（ブランチ）構成

- 各キャラ: 6~7路線 x 5スキル
- 各路線のSPコスト: 2 + 2 + 3 + 4 + 5 = **16SP/路線**
- 約100SPで5路線分（80SP）+ 余剰20SP
- 隠し路線（8つ目）: 後述

### スキル習得ルール

- 各路線のスキルは順番に習得（下位スキル未習得で上位不可）
- 路線間の前提条件なし（どの路線からでも着手可能）
- 習得済みスキルの変更は「記憶の書」でリセット

### SPコスト詳細（路線内）

| スキル段階 | SPコスト | 累計SP |
|-----------|---------|--------|
| Tier 1 | 2 SP | 2 |
| Tier 2 | 2 SP | 4 |
| Tier 3 | 3 SP | 7 |
| Tier 4 | 4 SP | 11 |
| Tier 5 | 5 SP | 16 |

---

## 3. 余剰SPの使い道

### ステータス強化

| 対象ステータス | SPコスト | 効果 |
|-------------|---------|------|
| ATK | 3SP | +1 |
| DEF | 3SP | +1 |
| INT | 3SP | +1 |
| RES | 3SP | +1 |
| AGI | 3SP | +1 |
| DEX | 3SP | +1 |
| LUK | 5SP | +1 |

- 各ステータス最大+10まで強化可能
- LUKのみコストが高い

### 隠し路線解放

- 条件: 余剰SP 50以上所持（消費ではなく所持チェック）
- 解放後、路線内16SPで全スキル取得可能
- 合計必要SP: 100（5路線） + 16（隠し路線） = 116SP以上
- NG+ボーナスが事実上必須

---

## 4. リセット

### 記憶の書

- アイテム「記憶の書」を使用でSPを全リセット
- 入手方法: ショップ購入（高額）/ イベント報酬（各章1個ずつ）
- 使用回数制限なし（所持数分）
- リセット対象: スキル習得のみ（ステータス強化もリセット対象）

### NG+リセット

- NG+開始時に全キャラのスキルツリーを自動リセット
- SPは前周取得分 + NG+ボーナスで再取得
- 累積ボーナス（+10SP/周）が加算される

---

## 5. パッシブスキルの扱い

- パッシブスキル: 習得した時点で常時発動
- スロット制限なし（習得済みパッシブは全て有効）
- アクティブスキルとパッシブスキルは路線内に混在
- パッシブの例: 「火耐性+10%」「クリティカル率+5%」「HP+10%」

## 9. 実装メモ

### データ構造

```
SkillTreeData
├── CharacterId
├── Branches[]
│   ├── BranchId
│   ├── BranchName
│   ├── IsHidden (bool)
│   ├── UnlockCondition (隠し路線の条件)
│   └── Skills[5]
│       ├── SkillId
│       ├── SkillName
│       ├── Tier (1-5)
│       ├── SpCost (2/2/3/4/5)
│       ├── IsPassive (bool)
│       └── SkillData (戦闘スキルデータへの参照)
└── StatBoosts[]
    ├── StatType
    ├── SpCost (3 or 5)
    └── CurrentLevel / MaxLevel
```

### ScriptableObject設計

- `SkillTreeSO`: キャラごとに1つ。路線とスキルの定義
- `SkillDataSO`: 既存。個別スキルの戦闘データ
- `HiddenSkillSO`: 隠しスキルの解放条件と効果
- `WeaponSkillSO`: 専用武器スキルのデータ

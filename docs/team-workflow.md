## Git ブランチ運用

### ブランチ構成
- `main`: リリース用。直接pushは**禁止**
- `develop`: 開発統合ブランチ。直接pushは**禁止**
- `feature/*`: 機能開発用（例: `feature/atb-battle-system`）
- `fix/*`: バグ修正用（例: `fix/hp-display-bug`）

### マージフロー
1. `feature/*` or `fix/*` ブランチで作業
2. **PR を作成して `develop` にマージ**（レビュー必須）
3. `develop` でテスト実施、バグがないことを確認
4. `develop` → `main` への PR を作成（PR本文に develop の PR番号を記載）
5. `main` にマージ

### 注意事項
- push する前に**必ずテストを実行**すること
- main / develop への直接 push は絶対にしない
- コミットメッセージは日本語OK、何をしたか分かるように書く
